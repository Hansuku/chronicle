# 地球编年史内容编辑

历史事件统一维护在 `content/events.json`，格式约束见 `content/events.schema.json`。

## 推荐编辑流程

1. 启动本地站点并访问 `/studio`。
2. 点击“连接内容文件”，选择本目录中的 `events.json`。
3. 从右侧内容库选择已有事件，或点击“新建”。
4. 在地图上点击落点；需要微调时直接拖动事件图钉。
5. 处理右侧校验错误后点击“保存事件”。
6. 点击“保存内容文件”，将整个内容包写回 `events.json`。

如果浏览器不支持直接写文件，“保存内容文件”会下载新的 `events.json`，再用它替换本目录中的同名文件即可。

## 数据口径

- `year` 是事件显示的唯一时间依据，必须为整数，且不会吸附到附近年份。
- `coordinates` 的顺序固定为 `[经度, 纬度]`。
- `editorialNote` 用来记录史学口径、歧义与待核实事项，不会显示在正式地图卡片中。
- 事件标注与疆域状态分开维护。政权边界或国号变化不能只通过事件图钉表达。
- 建议为重要事件补充 `sources`，每一项是一条书目、档案说明或网址。
